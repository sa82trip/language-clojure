지난 글에서 `내 함수` 를 만드는 법을 익혔다. 하지만 함수는 단순히 `입력 → 출력` 일 때만 쓸모 있는 게 아니다. 계산 중간에 `임시 이름` 을 붙이고, 상황에 따라 `갈래` 를 나누고 싶을 때가 있다. 이번에는 그 두 가지 — 로컬 바인딩(`let`)과 제어 흐름(`if` / `when` / `cond`)을 다룬다. 그리고 클로저를 쓰면서 `반드시` 한 번은 넘어지는 함정, `참/거짓의 진실(truthiness)` 까지.

아래 코드는 전부 `clj` 로 직접 실행한 진짜 결과다.


## let — 로컬 이름 붙이기 {#let-로컬-이름-붙이기}

`let` 은 `함수 안에서만 쓰는 임시 이름` 을 만든다. 중복 계산을 없애고, 코드를 읽기 좋게 만든다.

```clojure
(let [x 5
      y 10]
  (+ x y))
;; => 15

(let [radius 3
      area (* Math/PI radius radius)]   ; 위에서 정한 radius를 아래서 씀
  (str "넓이: " area))
;; => "넓이: 28.274333882308138"
```

`let` 의 바인딩은 `위에서 아래로` 흐른다. 두 번째 바인딩이 첫 번째를 참조할 수 있다(`area` 가 `radius` 를 씀). 자바의 `Math/PI` 처럼 자바 interop도 그대로 먹힌다.

`let` 안에서도 `구조 분해` 가 된다 — 4편의 맵 분해를 그대로 쓴다.

```clojure
(let [{:keys [name age]} {:name "정탐" :age 30}]
  (str name " (" age ")"))
;; => "정탐 (30)"
```

함수 매개변수가 아니더라도, 본문 중간에서 데이터를 풀어 쓰고 싶을 때 `let` + 분해 조합이 매일 등장한다.


## if — 기본 조건문 {#if-기본-조건문}

`(if 조건 참-branch 거짓-branch)` 형태. `else` 라는 키워드는 없다. 세 번째 인자(거짓 branch)는 생략 가능하다.

```clojure
(if true "진짜" "가짜")
;; => "진짜"

(if (< 3 5) "작다" "안 작다")
;; => "작다"
```

주의: `if` 의 각 branch에는 `식이 하나` 만 온다. 여러 줄을 실행하고 싶으면 `do` 로 감싸거나 `when` 을 쓴다.


## when — else가 필요 없을 때 {#when-else가-필요-없을-때}

`(when 조건 본문)` 은 `참일 때만` 실행하고, 거짓이면 `nil` 을 반환한다. `else` 가 필요 없는 단방향 조건에 쓴다. 본문에 `여러 식` 을 그대로 나열할 수 있어 부수효과가 여러 개일 때 편하다.

```clojure
(when (pos? 5)       ; 5는 양수 → 참
  (* 10 5))
;; => 50

(when (neg? 5)       ; 5는 음수 아님 → nil
  "안 됨")
;; => nil
```

규칙: `else` 가 필요하면 `if` , 필요 없으면 `when` . `if` 의 거짓 branch를 비워두는 것보다 `when` 이 더 솔직한 코드다.


## cond — 여러 갈래로 나누기 {#cond-여러-갈래로-나누기}

`(cond 조건1 결과1 조건2 결과2 ...)` . if-else 사다리를 한 번에. 마지막에 `:else` 를 두면 `기본값` 이 된다.

```clojure
(defn classify [n]
  (cond
    (neg? n)  "음수"
    (zero? n) "영"
    (< n 10)  "한 자리 양수"
    :else     "큰 수"))

(classify -3)   ;; => "음수"
(classify 0)    ;; => "영"
(classify 7)    ;; => "한 자리 양수"
(classify 100)  ;; => "큰 수"
```

`:else` 는 특별한 키워드가 아니라 `그냥 항상 참인 값` 일 뿐이다(아래 truthiness 참고). 관용적으로 `:else` / `:default` 를 쓴다.


## 참/거짓의 진실 — truthiness (가장 중요) {#참-거짓의-진실-truthiness--가장-중요}

이 절이 이 글의 핵심이다. 클로저에서 `거짓` 인 값은 `단 두 개` 뿐이다: `nil` 과 `false` . `나머지 전부` 는 `참` 이다.

```clojure
(if 0 "참" "거짓")     ;; => "참"   ← 0은 참!
(if "" "참" "거짓")    ;; => "참"   ← 빈 문자열도 참
(if [] "참" "거짓")    ;; => "참"   ← 빈 벡터도 참
(if nil "참" "거짓")   ;; => "거짓"
(if false "참" "거짓") ;; => "거짓"
```

`0` 이 참이라는 게 파이썬/C 출신에겐 충격적이다. 가장 많이 하는 실수:

```clojure
;; 빈 컬렉션인지 검사하려고 if를 쓰면?
(if [] "원소 있음" "비었음")   ;; => "원소 있음"  ← 틀렸다!
```

`[]` 는 `참` 이라서 잘못된 branch로 간다. 그래서 클로저에는 `빈 컬렉션 검사 관용구` 가 있다:

```clojure
(if (seq []) "원소 있음" "비었음")    ;; => "비었음"    ← 올바르다
(if (seq [1 2]) "원소 있음" "비었음") ;; => "원소 있음"
```

`seq` 는 빈 컬렉션에 `nil` , 원소가 있으면 시퀀스를 반환한다. `seq` 의 결과가 `nil` 인지 아닌지로 `비었음/안 비었음` 을 판정한다. `empty?` 도 있지만 `(seq coll)` 이 가장 관용적이고 일관적이다.


## if-let / when-let — 뽑으면서 판단하기 {#if-let-when-let-뽑으면서-판단하기}

값을 꺼내서, 그 값이 `참일 때만` 쓰고 싶은 경우가 매우 흔하다. 이걸 한 번에 하는 매크로가 `if-let` / `when-let` 이다.

```clojure
(if-let [n (:a {:a 1})]        ; n에 1 바인딩 → 참
  (str "찾았다: " n)
  "없다")
;; => "찾았다: 1"

(if-let [n (:b {:a 1})]        ; :b는 없음 → n이 nil → 거짓
  (str "찾았다: " n)
  "없다")
;; => "없다"

(when-let [user {:name "정탐"}]   ; user에 맵 바인딩 → 참
  (:name user))
;; => "정탐"
```

`(get map key)` 결과가 없으면 `nil` 이 되고, `if-let` 이 그걸 잡아 거짓 branch로 보낸다. 맵에서 옵션값을 안전하게 꺼낼 때 `반복적으로` 쓴다.


## 실전 조합 — let + if + 구조 분해 {#실전-조합-let-plus-if-plus-구조-분해}

지금까지 배운 걸 한 함수에 섞어보자. 역할에 따라 인사말을 바꾸는 함수:

```clojure
(defn greeting [{:keys [name role] :or {role "방문자"}}]
  (let [prefix (if (= role "admin") "관리자" "환영합니다")]
    (str prefix ", " name)))

(greeting {:name "정탐" :role "admin"})   ;; => "관리자, 정탐"
(greeting {:name "정탐"})                  ;; => "환영합니다, 정탐"
```

맵 분해(`:keys` + `:or` 기본값)로 입력을 풀고, `let` 으로 중간 결과(`prefix`)에 이름을 붙이고, `if` 로 갈래를 나눴다. 4편과 5편을 한 몸에 담은 모양새다. 클로저 코드의 대부분이 이 `템플릿` 의 변형이다.


## 10분 요약 {#10분-요약}

-   `let` : 함수 안의 임시 이름. 바인딩은 위→아래로 흐르고, 구조 분해와 같이 쓴다.
-   `if` : `(if 조건 참 거짓)` . branch엔 식 하나, `else` 키워드는 없다.
-   `when` : `(when 조건 본문)` . 거짓이면 `nil` , `else` 가 필요 없을 때 쓴다(본문 여러 식 가능).
-   `cond` : 여러 갈래. 마지막에 `:else` 로 기본값.
-   ****truthiness****: `nil` 과 `false` 만 거짓. `0` , 빈 문자열, `[]` 은 모두 `참` . 빈 컬렉션 검사엔 `(seq coll)` .
-   `if-let` / `when-let` : 값을 뽑아 바인딩하면서, 참일 때만 본문 실행.
-   실전 템플릿: 구조 분해(입력 풀기) → `let` (중간 이름) → `if` / `cond` (갈래) → 마지막 식(반환).

다음 글에서는 클로저로 `반복` 하는 법을 본다. `map` / `reduce` 로는 힘든, `명시적 반복` 이 필요한 순간에 쓰는 `loop/recur` 와 재귀까지. 거기까지 오면 시퀀스 함수 + 제어 흐름 + 반복으로, 어떤 알고리즘이든 클로저답게 풀어낼 수 있다.
