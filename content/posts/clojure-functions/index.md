지난 글까지 `map` / `filter` / `reduce` 로 시퀀스를 요리하는 법을 익혔다. 하지만 우리는 줄곧 `남이 만든 함수` 만 썼다. 이번에는 드디어 `내 함수` 를 직접 만든다. 함수를 정의하고(`defn`), 이름 없이 즉석에서 굴리고(익명 함수), 인자를 효율적으로 풀고(구조 분해), 인자 개수를 자유롭게 받는(가변 인자) 법까지. 이 네 가지면 클로저로 `문장` 을 쓰는 데 필요한 동사가 거의 다 갖춰진다.

아래 코드는 전부 `clj` 로 직접 실행한 진짜 결과다.


## defn — 이름 붙인 함수 {#defn-이름-붙인-함수}

가장 기본형. `defn 이름 [매개변수] 본문` 형태다. 본문의 `마지막 식` 이 반환값이다. `return` 이란 키워드는 없다.

```clojure
(defn greet [name]
  (str "안녕, " name "!"))

(greet "Clojure")
;; => "안녕, Clojure!"

(defn square
  "숫자의 제곱을 반환한다."   ; docstring (함수 설명)
  [x]
  (* x x))

(square 6)
;; => 36
```

이름 바로 뒤에 `문자열` 을 넣으면 `docstring` 이 된다. `(doc square)` 로 언제든 꺼내볼 수 있다. 클로저에서는 docstring을 `적극적` 으로 단다 — 함수가 `자기 설명서` 가 되기 때문이다.


## 여러 인자 개수 받기 — 다중 아리티(multi-arity) {#여러-인자-개수-받기-다중-아리티--multi-arity}

같은 이름의 함수를 `인자 개수(아리티)` 별로 여러 버전 정의할 수 있다. 자바/파이썬의 오버로딩과 비슷하지만, 한 몸 안에 들어있다.

```clojure
(defn add
  ([x] x)              ; 인자 1개 → 그대로
  ([x y] (+ x y))      ; 인자 2개 → 더하기
  ([x y z] (+ x y z))) ; 인자 3개 → 셋 다 더하기

(add 10)       ;; => 10
(add 10 20)    ;; => 30
(add 10 20 30) ;; => 60
```

각 아리티는 `([매개변수] 본문)` 모양의 괄호 쌍으로 묶는다. 인자 4개 이상이 필요하면? 그때 `가변 인자` 가 등장한다.


## 가변 인자 — `&` 로 남는 인자 모으기 {#가변-인자-and-로-남는-인자-모으기}

`&` 기호 뒤의 이름은 `나머지 인자 전체` 를 리스트로 모은다.

```clojure
(defn total [& nums]
  (apply + nums))

(total 1 2 3 4 5)
;; => 15

(total)
;; => 0
```

`& nums` 가 `1 2 3 4 5` 를 `(1 2 3 4 5)` 리스트로 묶는다. `+` 은 리스트를 못 더하니 `apply` 로 `풀어서` 넘긴다. `(apply + (list 1 2 3 4 5))` 는 `(+ 1 2 3 4 5)` 가 된다.

필수 인자 + 가변 인자 조합도 흔하다:

```clojure
(defn describe-person [name & hobbies]
  {:name name :hobbies hobbies})

(describe-person "정탐" "독서" "코딩" "산책")
;; => {:name "정탐", :hobbies ("독서" "코딩" "산책")}
```

`name` 은 첫 인자, `& hobbies` 가 나머지를 리스트로. `apply` 가 왜 가변 인자와 세트인지 감이 올 것이다.

```clojure
(apply max [3 1 4 1 5 9 2 6])   ;; => 9
(apply str ["a" "b" "c"])       ;; => "abc"
```


## 익명 함수 — 이름 없이 즉석에서 {#익명-함수-이름-없이-즉석에서}

두 가지 문법이 있다.

`fn` 폼: 가장 일반적. `map` 같은 곳에 즉석으로 넘기기 좋다.

```clojure
((fn [x y] (* x y)) 3 4)
;; => 12

(map (fn [x] (* x x)) [1 2 3 4])
;; => (1 4 9 16)
```

리터럴 `#(...)` : 짧은 익명 함수 전용. 매개변수를 `%1` , `%2` ... 로, 인자 하나면 `%1` 또는 그냥 `%` 로 쓴다.

```clojure
(map #(* % %) [1 2 3 4])
;; => (1 4 9 16)

(map #(+ %1 %2) [1 2 3] [10 20 30])
;; => (11 22 33)

(reduce #(+ %1 %2) [1 2 3 4 5])
;; => 15
```

둘 중 뭘 쓸까? `한 줄` 짜리면 `#(...)` 가 압도적으로 간결하다. `두 줄 이상` 이거나 `중첩` 될 때는 `fn` 이 읽기 편하다. 핵심 제약: `#(...)` 안에는 `하나의 식` 만 들어간다.


## 구조 분해 — 인자를 풀어서 받기 {#구조-분해-인자를-풀어서-받기}

컬렉션을 통째로 받지 않고, `그 안의 원소` 만 뽑아 이름을 붙이는 문법. 클로저 코드가 `간결한` 이유 중 하나다.


### 벡터 분해 {#벡터-분해}

매개변수 자리를 벡터 모양으로 적으면, `순서대로` 원소를 바인딩한다.

```clojure
(defn first-two [[a b]]
  [a b])

(first-two [10 20 30 40])
;; => [10 20]
```

`&` 와 섞으면 머리 하나 + 나머지:

```clojure
(defn head-and-rest [[first & others]]
  {:first first :rest others})

(head-and-rest [:a :b :c :d])
;; => {:first :a, :rest (:b :c :d)}
```


### 맵 분해 — 가장 강력하다 {#맵-분해-가장-강력하다}

맵의 키를 키워드로 꺼내 이름에 바인딩. `{:keys [이름들]}` 패턴이 실전의 90%다.

```clojure
(defn greet-user [{:keys [name age]}]
  (str name "은(는) " age "살"))

(greet-user {:name "정탐" :age 30 :city "서울"})
;; => "정탐은(는) 30살"
```

맵엔 `name` , `age` , `city` 가 있어도 내가 쓰는 것만 뽑는다. `city` 는 무시된다. 이게 클로저 코드가 `선언적` 인 비결이다 — "데이터에서 뭘 꺼낼지"를 `매개변수 자리` 에 적는다.

두 가지 장식을 더 알면 끝난다. `:or` 은 기본값, `:as` 는 원본 맵 전체를 보존한다.

```clojure
(defn format-post [{:keys [title author]
                    :or {author "익명"}   ; author 없으면 "익명"
                    :as post}]            ; 원본 맵 통째로도 받기
  (str "[" author "] " title " (키 " (count (keys post)) "개)"))

(format-post {:title "Clojure 함수" :author "정탐"})
;; => "[정탐] Clojure 함수 (키 2개)"

(format-post {:title "무명 글"})
;; => "[익명] 무명 글 (키 1개)"
```

둘째 호출엔 `author` 가 없지만 `:or` 덕에 `"익명"` 으로 채워지고, `:as post` 덕에 원본 키 수(1개)도 센다. 구조 분해 한 줄이 `if` 와 `(get ...)` 짜깍기 여러 줄을 대신한다.


## 10분 요약 {#10분-요약}

-   `defn 이름 [매개변수] 본문` : 이름 붙은 함수. 마지막 식이 반환값, `return` 없음. 이름 뒤 문자열은 `docstring`.
-   다중 아리티: 인자 개수별로 `([args] body)` 묶음을 나열. 오버로딩을 한 함수 안에.
-   가변 인자: `& rest` 가 남은 인자를 리스트로. `apply` 로 풀어서 넘기는 게 세트.
-   익명 함수: 한 줄이면 `#(...)` (`%1` / `%2` / `%`), 복잡하면 `(fn [x] ...)`. `#(...)` 안엔 식 하나만.
-   구조 분해: 벡터는 `[[a b]]` 순서대로, 맵은 `{:keys [a b]}` 키워드로. `:or` 기본값, `:as` 원본 보존.
-   `apply f coll` : `(f a b c ...)` 모양으로 `풀어서` 호출. 가변 인자 함수에 리스트를 넘길 때 필수.

다음 글에서는 함수 안의 `지역 이름(let)` 과 `제어 흐름(if/when/cond)` 을 다룬다. 여기까지 오면 데이터를 받아 → 풀고(구조 분해) → 판단하고(if) → 변환하여(함수) 돌려주는, `진짜 프로그램` 한 덩어리를 짤 수 있다.
